## 🛠️ مشاكل عامة

### 1. `dart:io` + Web
المشكلة الأكبر. عدة شاشات تستخدم `dart:io` (`File`, `Platform`) التي **لا تعمل على Flutter Web**:
- `payment_screen.dart` - `Platform.isIOS`
- `portfolio_gallery_screen.dart` - `File`
- `wizard_screen.dart` - `File`, `ImagePicker` (الـ Web يحتاج `cross_file`)
- `complaint_screen.dart` - `File`, `ImagePicker`

**الحل:** استخدام `universal_platform` package أو التحقق بـ `kIsWeb` من `foundation.dart`.

### 2. ألوان مباشرة بدلاً من Theme
في `account_management_screen.dart`:
- استخدام `Color(0xFF059669)` بدلاً من `AppColors.primary`
- استخدام `Colors.grey.shade200` بدلاً من `AppColors.border`

### 3. Mock Data في كل مكان
جميع الشاشات تقريباً تستخدم بيانات وهمية محلية بدلاً من API/Provider.

### 4. أزرار غير مفعلة
الكثير من الـ `onPressed: () {}` بدون أي إجراء:
- Dashboard: كل الأزرار
- Profile: مشاركة، مفضلة
- Account: تعديل، تبديل الوضع
- Notifications: تحديد الكل كمقروء
- Subscriptions: زر الاشتراك

### 5. نقص في Handling Errors
- `home_screen.dart`: لا يوجد error/empty state
- `artisan_profile_screen.dart`: لا يوجد error/loading state
- `account_screen.dart`: لا يوجد error/loading state
- `dashboard_screen.dart`: لا يوجد error/loading state
- `account_management_screen.dart`: لا يوجد loading state
- `subscriptions_screen.dart`: لا يوجد loading state

### 6. RTL مشاكل
بعض الشاشات:
- `home_screen.dart`: `textDirection` قد لا يكون معرفاً في كل مكان
- `artisan_profile_screen.dart`: بعض النصوص تستخدم `TextAlign` خاطئ
- `account_management_screen.dart`: لا يوجد `textDirection: TextDirection.rtl`
