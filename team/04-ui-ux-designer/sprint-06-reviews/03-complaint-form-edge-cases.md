## 4. الحالات الإضافية (Edge Cases)

### 4.1 إعادة تقييم (تعديل)

```dart
// المنطق:
// - المستخدم قيّم حرفياً (rating=4) → يريد تعديله
// - API: GET /api/v1/reviews/check
//   → has_reviewed: true, can_edit: true (خلال 7 أيام)
// - شاشة RV-02 (تعديل) تظهر مع البيانات الحالية
// - PUT /api/v1/reviews/:id → { rating: 5, comment: "..." }
// - بعد النجاح: ✅ "تم تعديل التقييم"

// تصميم الـ Edit:
// - نفس شاشة التقييم لكن:
//   - العنوان: "عدّل تقييمك" (بدلاً من "كيف كانت الخدمة؟")
//   - النصوص: "تقييمك الحالي: ⭐⭐⭐⭐"
//   - النجوم: قابلة للنقر
//   - زر: "حفظ التعديل"
```

### 4.2 حظر تقييم مكرر

```dart
// المنطق:
// - المستخدم يحاول تقييم نفس الحرفي لخدمة سبق تقييمها
// - API: GET /api/v1/reviews/check
//   → has_reviewed: true, can_edit: false (أكثر من 7 أيام)
// - اعرض شاشة المنع (Duplicate Prevention screen من 01-review-rating-screen.md)
// - إخفاء stars و textarea — فقط رسالة منع

// الـ Block على الـ Backend:
// - جدول reviews: UNIQUE (client_id, artisan_id, service_request_id)
// - إذا حاول POST ✗ → 409 Conflict → Client يعرض رسالة المنع
```

### 4.3 الـ Star Rating متوافق مع RTL

```dart
// مواصفات Flutter للـ Star Rating في RTL:
// 1. النجوم نفسها لا تنعكس (تبقى ⭐ من اليسار لليمين بصرياً)
// 2. استخدام Directionality.rtl للـ Row لا يؤثر على النجوم
// 3. التفاعل: النقر على النجمة الأولى (في RTL = أقصى اليمين) = 5 نجوم
//    - هذا غير بديهي. التوصية:
//    - في RTL: ترتيب النجوم يبقى 1-2-3-4-5 من اليسار لليمين
//    - لأن النجوم ليست نصاً — لا تحتاج للانعكاس الثقافي

// الحل:
// - Wrap النجوم في Row مع mainAxisAlignment.center
// - تجاهل Directionality للـ Row
// - النجوم دائماً 1 (يسار) → 5 (يمين) بالترتيب البصري
```

### 4.4 Push Notification للتقييم

```
┌──────────────────────────────────┐
│  🔔 Elmokef                      │
│  كيف كانت الخدمة مع يوسف العلوي؟│
│  اضغط لتقييم التجربة             │
│  منذ ساعتين                      │
└──────────────────────────────────┘

// توقيت الإشعار:
// - بعد 24 ساعة من اكتشاف الاتصال (إذا لم يقيّم المستخدم)
// - إشعار تذكيري بعد 48 ساعة (إذا لم يفتح)
// - إشعار أخير بعد 72 ساعة

// تنسيق الإشعار:
// Title: "Elmokef"
// Body: "كيف كانت الخدمة مع [اسم الحرفي]؟ اضغط لتقييم التجربة"
// Data payload: { "type": "review", "artisan_id": "123", "request_id": "456" }

// عند النقر:
// - إذا التطبيق مغلق → فتح → شاشة التقييم (RV-01)
// - إذا التطبيق في الخلفية → شاشة التقييم
// - إذا التطبيق مفتوح → Snackbar + فتح شاشة التقييم
```
