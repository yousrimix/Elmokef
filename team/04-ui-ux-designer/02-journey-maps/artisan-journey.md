**المرجع:** Persona 2 — حرّاث | User Stories US-12 إلى US-20

## الرحلة: حرّاث السباك يسجل في Elmokef

### المراحل:

```
[الاكتشاف] → [التسجيل (Wizard)] → [إعداد الملف] → [استقبال الطلبات] → [تقييم العملاء] → [التحسين]
```

---

## الخريطة التفصيلية

### المرحلة 1: الاكتشاف

| البعد | التفاصيل |
|-------|---------|
| **سيناريو** | حرّاث سمع من زميل له عن Elmokef — أو وصلت له مندوب ميداني |
| **القناة** | مندوب ميداني + إعلان فيسبوك + زميل حرفي |
| **العواطف** | 🤔 متشكك + 🧐 فضولي (هل هذا حقيقي؟) |
| **نقاط الألم** | - جرب قبل منصات وما عجبتهوش<br>- يخاف من النصب |
| **فرصة UX** | - عرض قصص نجاح حرفيين حقيقيين (فيديو قصير)<br>- شعار "مجاني — جرب ولا كتخسر والو" |

### المرحلة 2: التسجيل (Wizard)

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. فتح التطبيق ← 2. اختيار "أنا حرفي" ← 3. خطوة 1: الاسم والهاتف والعنوان ← 4. خطوة 2: اختيار الخدمات والأسعار ← 5. خطوة 3: رفع الصور ← 6. خطوة 4: اختيار الباقة ← 7. تأكيد |
| **العواطف** | 😊 متفائل في البداية ← 😟 قلق إذا طلب معلومات كثيرة ← 🧐 يقرأ كل خطوة بتركيز |
| **نقاط الألم** | - نموذج طويل يخوفه ويخليه يوقف في النص<br>- يطلب صور وهو ماعندوش صورمناسبة<br>- ما يفهمش بعض المصطلحات |
| **فرصة UX** | - Wizard بـ 4 خطوات مع Progress Bar<br>- كل خطوة فيها إرشادات بسيطة بالدارجة<br>- حفظ كمسودة (Draft) — يكمل لاحقاً<br>- مثال: "صوّر شغلك من هاد الزاوية" |
| **المس** | شاشة ترحيب → Step 1 → Step 2 → Step 3 → Step 4 → تأكيد |

### المرحلة 3: إعداد الملف

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. رفع صورة شخصية ← 2. كتابة وصف (خبرة، شهادات) ← 3. إضافة صور أعمال ← 4. معاينة الملف |
| **العواطف** | 🤔 غير واثق من صورته ← 😊 سعيد بعد معاينة الملف |
| **نقاط الألم** | - يخاف يحط صورته على النت<br>- ما عندوش صور شغل بجودة عالية |
| **فرصة UX** | - زر "شوف كيفاش باين ملفك للزبون" — معاينة مباشرة<br>- رسالة: "الحرفيين اللي عندهم صور كيجيهم 3 أضعاف الطلبات" |
| **المس** | إضافة صورة ← وصف ← صور أعمال ← معاينة |

### المرحلة 4: استقبال الطلبات

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. استقبال إشعار "طلب جديد" ← 2. فتح التطبيق ← 3. رؤية من اتصل به ← 4. الاتصال بالعميل |
| **العواطف** | 😊 سعيد ومتحمس ← 😟 قلق من عدم معرفة الرد |
| **نقاط الألم** | - ما يعرفش شنو يسوي أول ما يفتح التطبيق<br>- الإشعارات ما توصلش فالخلفية (خصوصاً على Huawei) |
| **فرصة UX** | - Dashboard بسيط: أول شيء هو "آخر من اتصل بك"<br>- زر واحد كبير: "اتصل بالزبون"<br>- نص إرشادي: "هاد الزبون قريب منك فالدار البيضاء" |
| **المس** | إشعار → Dashboard → ملف العميل → اتصال |

### المرحلة 5: التقييمات

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. إشعار "عليك تقييم جديد" ← 2. فتح التطبيق ← 3. قراءة التقييم ← 4. (اختياري) الرد |
| **العواطف** | 😊 فخور بالتقييم الجيد ← أو 😠 غاضب من تقييم سيء |
| **نقاط الألم** | - التقييم السيء بدون تعليق — ما يعرفش ليش<br>- قدرته على الرد محدودة (ما كتير قاري) |
| **فرصة UX** | - عرض التقييم بشكل بسيط: "النجوم + التعليق"<br>- زر "رد" بسيط: يكتب جملة قصيرة<br>- رسالة تحفيزية: "تقييمك 4.8 — أحسن من 90% من الحرفيين!" |
| **المس** | إشعار ← تقييم ← رد ← رجوع للـ Dashboard |

### المرحلة 6: التحسين

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. فتح لوحة الإحصائيات ← 2. مشاهدة عدد المشاهدات والاتصالات ← 3. تحسين ملفه |
| **العواطف** | 🧐 يريد فهم الأرقام ← 😊 إذا كانت إيجابية |
| **نقاط الألم** | - الأرقام والإحصائيات قد تكون معقدة<br>- ما يعرفش كيفاش يحسن ملفه |
| **فرصة UX** | - إحصائيات بصرية بسيطة (أيقونات + أرقام كبيرة)<br>- تلميحات ذكية: "زيد صور باش تزاد المشاهدات"<br>- Score اكتمال: "ملفك مكتمل 70%" |
| **المس** | Dashboard ← إحصائيات ← تحسينات ← معاينة |

## لحظات الحقيقة للحرفي

| اللحظة | التأثير | التصميم المقترح |
|--------|---------|----------------|
| **أول اشتراك** | 🟢 إذا سهل — يستمر. إذا صعب — يترك | 🟢 Wizard بـ 4 خطوات + حفظ كمسودة |
| **أول طلب** | 🟢 لحظة حاسمة — يثبت قيمة التطبيق | 🟢 إشعار فوري + اتصال مباشر بالعميل |
| **أول تقييم** | 🟠 يبني السمعة أو يحبطه | 🟢 عرض إيجابي + رد بسيط |
| **تجديد الاشتراك** | 🟠 نقطة قرار: هل يستمر أم لا؟ | 🟢 إظهار عدد الطلبات التي جاءت من التطبيق |

## توصيات تصميم خاصة بالحرفي (Simplified Mode)

1. **وضع المبسّط**: أزرار كبيرة (48dp+)، نصوص واضحة، أيقونات قليلة
2. **Wizard خطوات**: model → خطوة → تأكيد (ليس نموذج طويل)
3. **إرشادات بالدارجة**: "دير صورة واضحة وجهك باين" — وليس "Upload a profile photo"
4. **ألوان عالية التباين**: للحرفيين الذين قد تكون رؤيتهم ضعيفة
5. **دعم صوتي (للمرحلة الثانية)**: قراءة الإشعارات والتعليمات بصوت
