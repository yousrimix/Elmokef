# 5.2 — خريطة رحلة العميل (Client Journey Map)
**إعداد:** ليلى السعد — UI/UX Designer  
**المرجع:** Persona 1 — عميد | User Stories US-01 إلى US-11

---

## الرحلة: يبحث عميد عن سباك لإصلاح انسداد مغسلة

### المراحل:

```
[الاكتشاف] → [التسجيل] → [البحث] → [المقارنة] → [الاتصال] → [الخدمة] → [التقييم]
```

---

## الخريطة التفصيلية

### المرحلة 1: الاكتشاف (Awareness)

| البعد | التفاصيل |
|-------|---------|
| **سيناريو** | انسداد مغسلة في منزل عميد — يحتاج سباكاً فوراً |
| **القناة** | إعلان فيسبوك/إنستغرام + توصية من صديق |
| **العواطف** | 😠 منزعج + 😰 قلق من تكلفة الإصلاح |
| **نقاط الألم** | لا يعرف سباكاً موثوقاً، يخاف من المبالغة في السعر |
| **فرصة UX** | إعلان يوضح: "شوف السعر والتقييم قبل ما تكلم" |

### المرحلة 2: التسجيل (Registration)

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. تحميل التطبيق ← 2. فتح التطبيق ← 3. إدخال رقم الهاتف ← 4. إدخال OTP ← 5. إضافة الاسم |
| **العواطف** | 😊 متحمس لكن 😤 ينزعج إذا طلب بيانات كثيرة |
| **نقاط الألم** | يريد الوصول للخدمة بسرعة — أي نموذج طويل سيحبطه |
| **فرصة UX** | تسجيل بـ 3 خطوات فقط: رقم هاتف → OTP → اسم. الباقي لاحقاً |
| **المس** | شاشة ترحيب + إدخال رقم + OTP + إكمال بيانات |

### المرحلة 3: البحث (Search/Browse)

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. فتح التطبيق ← 2. رؤية فئات الخدمات ← 3. اختيار "سباك" ← 4. تفعيل الموقع ← 5. ظهور القائمة |
| **العواطف** | 😊 متحسن ← 😕 إذا طلب الموقع فوراً |
| **نقاط الألم** | - طلب الموقع مبكراً (قد يرفض الإذن) |
| **فرصة UX** | - تأجيل طلب الموقع لوقت الحاجة<br>- عرض الفئات بأيقونات + صور توضيحية<br>- إظهار "بدون موقعك، نعرض كل الحرفيين" |
| **المس** | شاشة رئيسية ← فئات ← قائمة مرتبة |

### المرحلة 4: المقارنة (Compare)

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. تمرير القائمة ← 2. النقر على حرفي ← 3. مشاهدة الملف ← 4. العودة للقائمة ← 5. مقارنة مع آخرين |
| **العواطف** | 🧐 فضولي + 😟 غير متأكد |
| **نقاط الألم** | - لا توجد طريقة سهلة للمقارنة بين حرفيين<br>- ينسا تفاصيل الحرفي الأول بعد مشاهدة الثاني |
| **فرصة UX** | - إظهار المعلومة الأساسية في البطاقة (سعر + تقييم + مسافة) بدون الحاجة لدخول الملف<br>- إضافة زر "مقارنة" يضيف الحرفي لقائمة مقارنة |
| **المس** | قائمة ← بطاقة حرفي ← ملف ← مقارنة |

### المرحلة 5: الاتصال (Contact)

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. اختيار الحرفي ← 2. الضغط على "اتصال" ← 3. اختيار وسيلة (واتساب/هاتف) ← 4. التحدث مع الحرفي |
| **العواطف** | 😊 متفائل + 😰 قلق من السعر النهائي |
| **نقاط الألم** | - السعر المعروض قد لا يكون السعر النهائي<br>- لا توجد ضمانات |
| **فرصة UX** | - عرض السعر التقديري بوضوح<br>- إضافة عبارة "السعر قابل للاتفاق" إذا كان السعر تقديرياً<br>- إظهار أن الحرفي "موثّق" (إذا اجتاز التوثيق) |
| **المس** | ملف حرفي ← نقر على واتساب/هاتف ← فتح التطبيق الخارجي |

### المرحلة 6: الخدمة (Service)

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. قدوم الحرفي ← 2. تنفيذ الخدمة ← 3. الدفع (نقداً أو عبر التطبيق في المستقبل) |
| **العواطف** | 😊 راضي ← أو 😠 غير راضي |
| **نقاط الألم** | - لا توجد متابعة من التطبيق بعد الاتصال |
| **فرصة UX** | - إشعار بعد 24 ساعة: "كيف كانت الخدمة؟"<br>- سؤال بسيط: "هل تمت الخدمة؟" (نعم/لا) ← يفتح التقييم |
| **المس** | خارج التطبيق (لكن إشعار بعد الخدمة) |

### المرحلة 7: التقييم (Review)

| البعد | التفاصيل |
|-------|---------|
| **الإجراءات** | 1. فتح الإشعار ← 2. اختيار عدد النجوم ← 3. كتابة تعليق ← 4. إرسال |
| **العواطف** | 😊 يحب المشاركة إذا كانت التجربة جيدة |
| **نقاط الألم** | - لا يوجد حافز للتقييم<br>- عملية التقييم معقدة (كثير من الحقول) |
| **فرصة UX** | - تقييم من خطوتين: النجوم أولاً ← (اختياري) تعليق<br>- عرض رسالة شكر بعد التقييم: "تقييمك ساعد 3 عملاء اليوم!" |

## ملخص لحظات الحقيقة (Moments of Truth)

| اللحظة | التأثير | التصميم المقترح |
|--------|---------|----------------|
| **طلب الموقع** | 🔴 إذا طلب مبكراً — قد يرفض ويغلق | 🟢 اطلبه عند الحاجة وليس في البداية |
| **عرض الأسعار** | 🟢 إذا ظهر السعر بوضوح — يزيد الثقة | 🟢 بطاقة الحرفي تعرض السعر من أول نظرة |
| **المقارنة** | 🟠 حالياً غير موجودة — يضيع العميل | 🟢 إضافة Compare mode |
| **بعد الخدمة** | 🟠 لا متابعة — العميل ينسى التقييم | 🟢 إشعار بعد 24 ساعة مع رابط تقييم سريع |

---

## فرص التحسين (Quick Wins)

1. ⚡ **بطاقة الحرفي** — اجعل السعر والتقييم والمسافة ظاهرين دون الحاجة للنقر
2. ⚡ **شاشة الخدمات** — استخدم صوراً حقيقية للفئات (ليست أيقونات فقط)
3. ⚡ **تسجيل سريع** — 3 خطوات فقط: رقم → OTP → اسم
4. ⚡ **إشعار ما بعد الخدمة** — يزيد التقييمات بنسبة تقديرية 40%
