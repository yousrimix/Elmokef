### أنواع أخطاء الدفع

| رمز الخطأ | الرسالة | الإجراء |
|-----------|---------|---------|
| `insufficient_funds` | البطاقة غير كافية أو تم رفضها | جرب بطاقة أخرى |
| `card_expired` | البطاقة منتهية الصلاحية | اختر بطاقة أخرى |
| `card_declined` | تم رفض البطاقة من البنك | اتصل بالبنك + جرب بطاقة أخرى |
| `network_error` | فشل الاتصال بخادم الدفع | إعادة المحاولة |
| `timeout` | انتهت مهلة الدفع | إعادة المحاولة |
| `cmi_error` | خطأ من CMI | جرب لاحقاً، اتصل بالدعم |
| `general_error` | حدث خطأ غير متوقع | جرب لاحقاً |

## 6. الـ API Endpoints

| الطريقة | الـ Endpoint | الاستخدام |
|---------|-------------|-----------|
| POST | `/api/v1/subscriptions/subscribe` | `{plan_id, payment_method}` → redirect_url لـ CMI |
| POST | `/api/v1/subscriptions/cancel` | إلغاء الاشتراك |
| GET | `/api/v1/subscriptions/status` | الحالة الحالية |
| POST | `/api/v1/subscriptions/retry-payment` | إعادة محاولة دفع معلق |
| Webhook | `POST /api/v1/payments/cmi-webhook` | استقبال تحديثات CMI |

### تنسيق Webhook CMI

```json
{
  "transaction_id": "cmi_123456",

  "paid_at": "2026-09-15T10:30:00Z",
  "signature": "hash_..."
}
```
