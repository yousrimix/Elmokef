## 1. اشتراكات (7 حالات)

### TC-SUB-01: مشاهدة الباقات (Free/Pro/Premium)
| الحقل | القيمة |
|-------|--------|
| الشاشة | Subscription Screen |
| الخطوات | 1. فتح شاشة الاشتراكات 2. عرض الباقات الثلاث |
| المتوقع | 3 بطاقات: Free (0 DH), Pro (سعر), Premium (سعر) — كل بطاقة تعرض المميزات |
| التركيز | Failure scenario: لا شيء (مجرد عرض) |
| الحالة | ⬜ |

### TC-SUB-02: اشتراك Free بدون دفع
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. اختيار باقة Free 2. تأكيد الاشتراك |
| المتوقع | اشتراك فوري (لا دفع), Badge "حالي" يظهر, الحالة = active |
| الحالة | ⬜ |

### TC-SUB-03: اشتراك Pro/Premium → WebView CMI
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. اختيار Pro 2. تأكيد الدفع 3. ملاحظة فتح WebView |
| المتوقع | WebView يفتح على بوابة CMI مع المبلغ الصحيح ومعرّف العملية |
| الحالة | ⬜ |

### TC-SUB-04: إلغاء اشتراك
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. إلغاء اشتراك حالي 2. التحقق |
| المتوقع | حالة الاشتراك = cancelled, تاريخ الإنهاء = نهاية الدورة الحالية |
| الحالة | ⬜ |

### TC-SUB-05: ترقية من Pro إلى Premium
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. مشترك في Pro 2. اختيار Premium 3. دفع فرق السعر |
| المتوقع | فرق السعر = Premium - Pro (pro-rated), الترقية فورية بعد الدفع |
| الحالة | ⬜ |

### TC-SUB-06: اشتراك من حساب عميل (ليس حرفي)
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. تسجيل الدخول كعميل 2. محاولة فتح شاشة الاشتراكات |

| الحالة | ⬜ |

### TC-SUB-07: مشاهدة الاشتراك الحالي
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. حرفي لديه اشتراك نشط 2. فتح شاشة الاشتراكات |
| المتوقع | الباقة الحالية مميزة بـ Badge "حالي", تظهر تفاصيل الاشتراك (تاريخ البدء والانتهاء) |
| الحالة | ⬜ |

## 2. دفع عبر CMI (8 حالات)

### TC-PAY-08: دفع ناجح — بطاقة صحيحة
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. إدخال بطاقة صحيحة في WebView CMI 2. إتمام الدفع |
| المتوقع | WebSocket تأكيد, الاشتراك يُفعّل فوراً, إشعار "تم الدفع بنجاح", تحديث الـ UI |
| الحالة | ⬜ |

### TC-PAY-09: رفض البطاقة (Insufficient funds)
| الحقل | القيمة | الفشل |
|-------|--------|-------|
| الخطوات | 1. إدخال بطاقة بدون رصيد 2. محاولة الدفع |
| المتوقع | WebView يعرض رسالة رفض من CMI, زر "إعادة المحاولة" يظهر, لا تغيير في حالة الاشتراك |
| السيناريو | ⚠️ **فشل — بطاقة مرفوضة** |
| الحالة | ⬜ |

### TC-PAY-10: إلغاء الدفع من WebView
| الحقل | القيمة | الفشل |
|-------|--------|-------|
| الخطوات | 1. فتح WebView 2. إلغاء العملية يدوياً (الرجوع أو إغلاق WebView) |
| المتوقع | العودة لشاشة الباقات, لا تغيير في الاشتراك, رسالة "تم إلغاء الدفع" |
| السيناريو | ⚠️ **فشل — إلغاء من المستخدم** |
| الحالة | ⬜ |

### TC-PAY-11: Timeout في WebView
| الحقل | القيمة | الفشل |
|-------|--------|-------|
| الخطوات | 1. فتح WebView 2. عدم إدخال بيانات والانتظار حتى timeout (دقيقتان) |
| المتوقع | رسالة "انتهت المهلة، حاول مرة أخرى", WebView يُغلق, الاشتراك لا يتغير |
| السيناريو | ⚠️ **فشل — Timeout** |
| الحالة | ⬜ |

### TC-PAY-12: Webhook تكراري (Idempotency — Replay)
| الحقل | القيمة | الفشل |
|-------|--------|-------|
| الخطوات | 1. إرسال webhook بنفس transaction_id مرتين متتاليتين |
| المتوقع | المرة الأولى: 200 + تفعيل. المرة الثانية: 200 (وليس 500) — Idempotency, لا ازدواجية في الاشتراك |
| السيناريو | ⚠️ **فشل — Replay Attack** |
| الحالة | ⬜ |

### TC-PAY-13: تجديد تلقائي ناجح
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. اشتراك مع تجديد تلقائي 2. وصول تاريخ التجديد 3. دفع تلقائي ناجح |
| المتوقع | اشتراك جديد + إشعار "تم تجديد اشتراكك", حالة الاشتراك = active |
| الحالة | ⬜ |

### TC-PAY-14: تجديد تلقائي فاشل
| الحقل | القيمة | الفشل |
|-------|--------|-------|
| الخطوات | 1. اشتراك مع تجديد تلقائي 2. البطاقة منتهية الصلاحية 3. فشل الدفع التلقائي |
| المتوقع | إشعار "فشل التجديد، قم بتحديث وسيلة الدفع", تخفيض الاشتراك إلى Free بعد 3 أيام |
| السيناريو | ⚠️ **فشل — بطاقة منتهية** |
| الحالة | ⬜ |

### TC-PAY-15: Audit Log للمدفوعات
| الحقل | القيمة |
|-------|--------|
| الخطوات | 1. تنفيذ عمليات دفع متنوعة (نجاح، فشل، إلغاء) 2. التحقق من Audit Log |
| المتوقع | كل عملية مسجلة: timestamp, user_id, amount, status, transaction_id, IP |
| الحالة | ⬜ |
